# StreamDeck Remote

Mobile controller for the Windows `streamdeck-clone` app. The desktop app owns the profiles, buttons, and action execution; this app wakes the PC, connects to the desktop WebSocket server, mirrors the active deck config, and sends button trigger commands.

## Desktop Contract

The Windows app should expose:

- WebSocket host: `0.0.0.0`
- WebSocket port: `37123`
- Incoming messages: `getConfig`, `triggerButton`
- Outgoing config message: `{ "type": "config", "config": DeckConfig }`

That matches the current `streamdeck-clone` Rust/Tauri bridge.

## Native Dev Build

`react-native-udp` is a native module, so Wake-on-LAN needs a custom Expo dev client or native build. This repo now includes `expo-dev-client`, an Android native project generated by Expo prebuild, and an EAS development profile.

The generated `android/` folder exists locally after prebuild. The existing Expo ignore rules still keep native folders out of normal git status, so run `npm run prebuild:android` again if you regenerate the workspace from source.

Common Android flow:

```bash
npm run android:dev
npm run start:dev-client
```

Cloud/internal dev build flow:

```bash
eas build --profile development --platform android
npm run start:dev-client
```

## Wake-on-LAN

Wake-on-LAN sends a UDP magic packet to the configured MAC address. It requires:

- Wake-on-LAN enabled in BIOS/UEFI.
- Wake-on-LAN enabled on the Windows network adapter.
- The phone and PC on the same LAN or routed broadcast support.
- A native Expo/dev build that includes `react-native-udp`. Expo Go can connect to the desktop app, but it cannot send UDP packets.

## Windows Sign-In Keyboard

The Android native build includes a manual Bluetooth HID keyboard mode:

1. Open the Android native dev build.
2. Tap **Enable** in the Phone Keyboard panel.
3. Pair the phone with Windows from Bluetooth settings. Windows should see it as a keyboard named `StreamDeck Remote Keyboard`.
4. Wake the PC.
5. Use the keypad in this app to enter your Windows PIN manually.
6. After Windows signs in and the desktop StreamDeck app starts, tap **Connect** in this app.

This does not store the PIN and does not auto-type it. It only sends individual key presses you tap.

Important Android limits:

- Android 9+ is required for the Bluetooth HID Device APIs.
- The phone hardware/ROM must support Bluetooth HID Device mode.
- Android may unregister the HID app if it is not foreground.
- Full reboot reconnect behavior depends on the PC Bluetooth adapter and Windows pairing state.

This app does not store or auto-type the Windows PIN. The Bluetooth keyboard mode works because Windows sees the phone as an external keyboard; the desktop StreamDeck server still cannot control the secure login desktop before sign-in.

## Desktop Startup

For automatic availability after sign-in, register the Windows `streamdeck-clone` app as a startup app. The quick Windows path is to add a shortcut to `shell:startup`; the product-code path is to add Tauri autostart support in the desktop project. This mobile repo only controls the remote side.
